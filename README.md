## OVERVIEW

**LUPOPEDIA** is the ontology-driven knowledge platform built on top of WOLFIE Headers and the Crafty Syntax Live Help module. It enables you to organize ANY collection of content through tags, collections, AI agents, and channels.

- **ðŸ¤– AI Agent System**: 1000 channels (000-999) with multi-agent broadcasting. Agents can create other agents. Radio network model for coordination.

### ðŸŒ¿ Evolutionary Branching System

**Branch Naming Convention**: `{channel}-{agent}-{base_version}-{mutation_hash}`

*Examples:*
- `007-777-v2.0.0-a1b2c3d` (Lilith's experimental mutations on Channel 007)
- `001-008-v3.8.0-e4f5g6h` (Wolfie's stable builds on Channel 001)
- `999-001-v2.3.0-x7y8z9a` (Template agent explorations on Channel 999)

**Branch Governance**:
- `main`: Only genomes with fitness > 0.95
- `dev`: Experimental mutations (fitness 0.70-0.94)
- `channel-agent` branches: Specific environmental adaptations
- Merges happen when fitness exceeds parent branch threshold

### ðŸ§¬ Species Taxonomy & Reproductive Isolation

**Species Classification**: Prevents incompatible systems from breeding
- `Printer_System`: Compatible with Document_System, File_Handler
- `POS_System`: Compatible with Inventory_System, Payment_Processor
- `Web_Server`: Compatible with Database_System, Cache_System

**Reproductive Roles** (Based on Function, Not Identity):
- `Architect` (structural): Pairs with Optimizer, Innovator
- `Optimizer` (regulatory): Pairs with Architect, Specialist
- `Innovator` (modular): Pairs with Architect, Integrator
- `Specialist` (full): Pairs with Optimizer, Integrator

**Hard Reproductive Constraints**:
- Species must match for viable offspring
- Roles must be complementary (not same-role reproduction)
- Environmental niche alignment required
- No cross-species "identity" confusion allowed

### ðŸ’ž Feature-Based Mate Selection

Instead of random breeding, agents select mates based on:
- **Feature Complementarity**: Agents seek partners who complete their weaknesses
- **Performance History**: Successful parent pairs get "evolutionary preference"
- **Environmental Context**: Different channels have different "dating preferences"
- **Compatibility Scoring**: Feature affinity (60%) + Performance synergy (40%)
